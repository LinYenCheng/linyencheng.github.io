Chrome Audit

把 chrome 的開發者工具打開後，Audits 頁籤可以進行幾個項目的測試，測完之後會有一些建議，大概看起來初步要做的就是增加快取的可用性~

Cache

查了一些快取相關資料後，發現這篇關於cache的寫得很詳細，另外常用又不太需要更改的函式庫改成使用 CDN 取得，如果有使用 Webpack 的話需要把這些函式庫加入 externals 還有透過 ProvidePlugin 給命名，就可以正常使用，有個小問題就是像是 lodash 這種函式庫建議不要使用 async 的標籤，要不然就是用到什麼 import 什麼，減少一次整包那樣~
