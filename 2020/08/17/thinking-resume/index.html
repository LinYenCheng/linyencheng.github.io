履歷診斷和優化履歷三步驟 職場老鳥必點技能

優化履歷前要先懂診斷履歷，履歷的好壞簡單說就是和 JD (Job Desription) 的符合程度，當履歷符合市場上的需求之後，接著就是依照履歷的受眾來進行相關優化。

履歷診斷

診斷履歷的五大重點:

  1. 履歷樣式
  2. JD 符合程度
  3. 相關經驗
  4. 時間資訊
  5. 履歷非相關資訊

履歷樣式

  1. 排版簡單不要分好幾欄，目的是方便機器閱讀
  2. 顏色全部都黑色統一字型即可，印表機不一定有彩色

JD 符合程度

可以快速查看 JD 中關鍵字在履歷中是否有出現? 建議履歷撰寫以相關經驗為主，即使是轉職只有實習、進修培訓也要放，剛畢業則可以列修過哪些相關的課。非相關經驗不需佔太多版面，列了並不會加分。

相關經驗

  1. 每一個工作經驗中建議都簡單介紹公司或部門，因為其他人不一定會知道
  2. 專案簡單介紹，若過於詳細可以列在作品集
  3. 每個工作經驗列出對應成就，責任不用多列
    • 責任: 負責什麼，因為每個人加入這份工作都應該要做
    • 成就: 可量化的或是有沒有得獎，同樣的一個人跟你做同個職缺，在哪些方面可以做得比別人更好
    • 目標: 告訴對方過去如何解決問題及對團隊帶來幫助，為了是預測未來可能可以帶來的幫助
  4. 相關學歷，如果碩士論文的內容跟這個工作有相關，建議補上 bullet point

時間資訊

  1. 工作超過 15 年，寫 15 年以上，不需要讓人感覺太老
  2. 工作時間太短可以只寫年份
  3. 畢業年份放結束即可，不需要被知道延畢幾年

履歷非相關資訊

  1. 興趣不需要寫適合面試的時候聊
  2. 列出參加相關研討會、讀書會
  3. 相關的 Side Project
  4. 相關社團經驗

履歷優化

在職涯初期通常履歷需要針對職缺 JD 優化，職涯後期才可能有企業為你設置一個職位。履歷的受眾主要分成搜尋演算法、用人單位 (HR 或用人主管)、朋友和獵頭，所以我們可以從三個方向去優化履歷:

  1. 針對 JD 關鍵字優化
  2. 針對 JD 要求優化
  3. Linkedin 優化

針對 JD 關鍵字優化

首先要讓履歷可以透過演算法被搜尋出來，通常 HR 就是依照 JD 上的內容協助進行搜尋，標題跟內文關鍵字一定要做 SEO 讓篩選系統篩選用，履歷排版建議簡單不要分好幾欄，方便機器閱讀。

針對 JD 提供對應說明

第二個 TA 是可能是對履歷沒有耐心的 HR，這時候就需要針對 JD 要求優化說法，會做事情同時也要會行銷自己，舉例來說正常的前端工程師職位，會需要:

  • 人才熟悉相關的語言和技術
  • 需要處理整個開發流程，包含設計、開發、測試、發佈
  • 與非技術相關人員的溝通的經驗

行銷自己的相關說法:

  • 若專案為一人維護，則可以強調有獨立作業能力，從設計、建置到測試都能一手包辦
  • 維護可以改成厲害一點的說法，像是持續監控與改善程式的品質與效能
    • 改善了多少效能，也許可以在工作時就將相關紀錄記下來
    • 透過什麼樣的方式增加了品質
  • 使用 CI/CD 工具則可以說明透過加強 Jenkins 或 Gitlab CI 的使用來優化產品的發佈流程
  • 若是有經營部落格或是開線上課都可以寫進履歷裡
    • 像是我寫部落格，是希望透過淺顯的文字分享六年多 Web 開發經驗，這裡也可以補上多少瀏覽數或有哪些熱門文章

Linkedin 優化

會看到 Linkedin 通常是人力仲介或是獵頭，也可能是學長姊、同學、朋友等弱連結。

版面一進來，看到的會是簡介，這個部分主要是介紹自己是誰，我的簡介如下:

Web 前後端開發經驗六年，一開始是後端工程師，五年前決定和瀏覽器一起發展而進入前端工程領域，專注在 React.js、 Next.js 相關技術及應用上，後端上使用 Node.js、Socket.IO 做 API 開發和即時資料交換。

介紹自己後，同時就要選對應的幾個技能放在 Skill Set 裡面，只需要放與個人相關事情就好，像是溝通能力這種無法證明的軟實力或是寫作、簡報這種基礎能力可以不用寫。

善於溝通

Skill Set 的區塊可以優化的三個部分:

  1. 要和簡介有一致性
  2. 可以找大家幫忙按讚，選了之後建議少換順序
  3. 列出來的技能要跟求職相關

Linkedin 四個可以優化的部分:

  1. 建議放專業的背景圖
  2. 大頭照不要放畢業照或是黑白套裝照看起來很菜
  3. 可以告訴其他人想要找怎麼樣的工作
  4. 在哪個地點建議不用寫，可能會被看成是外國人
